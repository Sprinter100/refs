Пуш.
  git push
Что такое фастфовард?
  Слияние веток без создания коммита слияния.
  Возможно только если в ветке, в которую идёт слияние, нет новых коммитов.
  При фаст форварде не возможны конфликты.
Чем опасен пуш с форсом?
  Тем, что данные на сервере перезаписываются.
Ветвление. Как создать и удалить ветку, локально и в origin`е.
  Создание:
  git checkout -b <branch-name>
  git push <remote-name> <branch-name>
  Удаление:
  git branch -d (-D если ветка не была слита с другой) <branch-name>
  git push origin --delete <branch-name>
Как привязать локальную ветку "a1" к удаленной ветке "b2", и пушить в нее?
  git branch -u origin/<remote_branch> - привязывает лок. ветку у удал.
  git co -b <local_branch_name> origin/<remote_branch> - Создает новую ветку от удал-й
  Как пушить:
  Если имена локальной и удал-й веток одинаковые - git push
  Если имена разные:
  1 способ
    git config --global push.default upstream - пушить в удал. ветку, привязанную к лок-ной.
    Затем git push
  2 способ
    git push -u origin <local_branch>:<remote_branch>
Как получить список комитов из локальной ветки, которую вы только что удалили?
  Через git reflog найти SHA1 последнего коммита удалённой ветки
  Затем git log <SHA1 удалённой ветки>

что такое мердж?
  Слияние веток.
  git merge <other_branch_name>
  git merge --no-ff - Создать коммит мерджа, даже если можно сделать фаст форвардом.
Как его откатить?
  Если мердж не закончен - git merge --abort
  Если мердж без фаст форварда - git reset --hard HEAD~1
  Если с фф - git reset --hard <sha1 коммита основной ветки до слияния>
Какие бывают стратегия мерджа?
  -Xignore-all-space
  -Xignore-space-change

Как удалить последний?
  git reset HEAD~1 - удаляет последний коммит, изменения становятся неиндексированными
  git reset --soft HEAD~1 - удаляет последний коммит, изменения индексированны
  git reset --hard HEAD~1 - удаляет последний коммит, изменения пропадают.
